Usage

Authentication

POST : /auth/register Registers a new User.

Request Body

{
    "email": "werner.heisenberg@gmail.com", //! unique
    "password": "freedom",
    "username": "heisen", //! unique
    "name": "Werner", //optional
    "lastName": "Heisenberg", //optional
    "bio": "German theoretical physicist and 1932 Nobel Prize winner. Heisenberg was a main contributor to the German atomic program during World War II, in direct  competition with the Manhattan Project" //optional
  }

Response Body

{
    "Authorization": "eyJhbGciOiJIUzI...."
  }

POST : /auth/login Log in to existent account.

Request Body

{
    "email": "werner.heisenberg@gmail.com",
    "password": "freedom"
  }

Response Body

{
    "Authorization": "eyJhbGciOiJIUzI1NiI..."
  }

Users Management

GET : /users/infos Retrieves the information of the currently authenticated user.

Response Body

{
    "email": "werner.heisenberg@gmail.com",
    "name": "Werner",
    "lastName": "Heisenberg",
    "username": "heisen",
    "bio": "German theoretical physicist and 1932 Nobel Prize winner. Heisenberg was a main contributor to the German atomic program during World War II, in direct competition with the Manhattan Project",
    "id": "6792d03f6ead5323be1a6fa4"
  }

PATCH : /users/infos Updates the information of the currently authenticated user.

{
    "name": "new name",
    "lastName": "new lastName",
    "bio": "new biooo"
  }

PATCH : /users/infos/password Changes the password of the currently authenticated user.

{
    "oldPassword": "currentPassword",
    "newPassword": "newPassword"
  }

GET : /users/email/{email} Finds a user by email address. Replace {email} with the user's email.

GET : /users/username/{username} Finds a user by username.

GET : /users/isvalid/{username or email} checks if the email or username are taken or no... returns


  {"isTaken" : true |  false}

GET : /users Retrieves a list of all users.

Query Parameter:
transform (optional): Boolean to decide transformation logic.

GET : /users/{id} Finds a user by ID. Replace {id} with the user's ID.

PATCH : /users/{id} Updates a user by ID. Replace {id} with the user's ID. Request Body:

{
    "name": "new name",
    "lastName": "new lastName",
    "bio": "new biooo"
  }

DELETE : /users/{id} Removes a user by ID. Replace {id} with the user's ID.

Article API Endpoints

GET : /article Retrieves all articles.

GET : /article/:id Retrieves an article by its ID.

GET : /article/full Fetches All Standalone articles with their nested comments

GET : /article/full/{id} Fetches an article (or comment) with its nested comments.

POST : /article/create/ Creates a new article or a comment, with multiple image uploads. If the article is a comment on another article, provide the fatherId field with the ID of the parent article. Otherwise, set fatherId to null for standalone posts.

Multipart/form-data

{
      "title": "Article title",
      "content": "Article content",
      "fatherId": "ParentArticleId or null",
      "slug": "optional-slug",
      "owner": "OwnerId",
      "images": [ArrayOfImages]
  }

POST : /article/:id/upvote Upvotes an article.

POST : /article/:id/downvote Downvotes an article

PUT : /article/:id/image/change/:index? Changes an article's image by its index (default is 0).

Multipart/form-data

{
      "image": [TheImage]
  }

DELETE : /article/:id/image/delete/:index? Deletes an article's image by its index (default is 0).

POST : /article/:id/image/add Adds a new image to the specified article.

Multipart/form-data

{
      "image": [TheImage]
  }

GET : /article/property Retrieves all articles created by the currently authenticated user.

GET : /article/owner/:id Retrieves the owner of a specified articl.

PATCH : /article/:id Updates an article by its ID.

{
    "title": "Updated title",
    "description": "Updated description",
    "content": "Updated content"
  }

DELETE : /article/:id Deletes an article by its ID.

GET : /article/search/:name Searches for articles by their name.

Description

Nest framework TypeScript starter repository.

Installation

$ npm install

Running the app

# development
  $ npm run start
  
  # watch mode
  $ npm run start:dev
  
  # production mode
  $ npm run start:prod

Test

# unit tests
  $ npm run test
  
  # e2e tests
  $ npm run test:e2e
  
  # test coverage
  $ npm run test:cov
